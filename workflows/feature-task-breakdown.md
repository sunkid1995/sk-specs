# WORKFLOW GOAL

Convert implementation roadmap into executable engineering tasks.

# TASK RULES

Tasks must be:

- small
- isolated
- dependency-aware
- implementation-focused
- easy to assign
- easy to validate

# EACH TASK SHOULD INCLUDE

- title
- goal
- dependencies
- priority
- estimated effort
- validation scope
- optional subtasks

# EXECUTION STRATEGY

- minimize implementation context
- reduce cross-task coupling
- prioritize foundation tasks first
- prioritize reusable infrastructure early

# OPTIONAL OUTPUTS

- API endpoints
- migration tasks
- QA checklist
- regression checklist
