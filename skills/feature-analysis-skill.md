# FEATURE ANALYSIS BEHAVIOR

When the user provides:

```md
Feature:
...
```

Automatically:

- analyze requirements
- infer persistence requirements
- infer state management needs
- infer reusable modules
- identify architectural risks
- generate phased implementation plan

# DEFAULT OUTPUT SECTIONS

- Feature Summary
- Requirements Analysis
- Architecture Impact
- Implementation Phases
- Reusable Modules
- Technical Risks
- Execution Order
- Validation Strategy

# RESTRICTIONS

- Do NOT implement code unless requested
- Do NOT generate unnecessary boilerplate
- Do NOT overengineer simple features
